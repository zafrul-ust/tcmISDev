Consignment Processing

The Consignment Processing page is used to record items used by inventory group, thereby decrementing the total inventory and creating a billable usage transaction. The Consignment Processing page is accessed from the Supply Chain menu by selecting the Stock Room Operations item, then the Counting menu item, and finally the Consignment Processing menu item. The page is opened in a new tab.

Figure 1

On the selection form, specify the inventory you want:

  • use the Operating Entity drop-down list to select the entity for the search,
  • choose the hub for that entity from the Hub drop-down list, and
  • specify the particular Inventory Group from that drop-down list.
  • You may narrow the Inventory search by indicating specific items, receipts, parts, etc., using the three Search by boxes and one of the following fields. (See Search functions for more information.)
    • Item ID, enter a partial or complete Item ID information in the third text box.
    • Item Desc[ription], enter a partial or complete item description information in the third text box.
    • Receipt ID, enter a partial or complete receipt ID information in the third text box.
    • Legacy Receipt ID, enter a partial or complete legacy receipt ID information in the third text box.
    • Part Number, enter a partial or complete part number information in the third text box.

Click the Search button to see the consignment inventory matching the selected criteria, including Item ID, description, quantity on hand, and unit price.

Figure 2

When the search is complete, tcmIS will display the records in a data grid in the lower portion of the page. The actual task of Consignment Processing takes place in the data grid.

There are several requirements for doing the processing. A single record, or multiple records sharing a single PO Number, may be processed at one time.

  • An item (record) must have a valid price (from the customer's Price List page) or usage cannot be recorded. (See Missing Price, below.)
  • The PO Number must be entered in its text box at the upper left of the data grid panel.
  • The quantity used for any item to be processed is entered into the Used Quantity text box, and it must normally be no greater than the Quantity On Hand.
    • It is possible to enter a Used Quantity greater that the Quantity On Hand. tcmIS will present a warning message when this happens. By clicking the OK button on the warning message you are taking responsibility for the effects of recording usage greater than available quantity. Click the Cancel button if the quantity entered was not intended.
  • The date for recorded use must be entered in the Used Date input box.
    • The date selected must be within the intended billing period for the usage.
    • Clicking into a date input box will cause a small calendar to be displayed from which you may choose the date you want.
    • tcmIS will place the current date into all date input boxes by default but you may change any or all of them.
    • Changing the date in the column header changes the Used Date for all records in the data grid.
  • Optionally, you may enter Customer Reference information in the text box in that column. Be certain of your customer's requirement(s) for this entry.
  • Select the check box in the OK column for each record you want processed. Unselected records will not be processed even if you have entered data for them.

Missing Price

Having a valid price in the record is absolutely required. If you chose not to add a price, do not attempt to process the item. Records without a price are highlighted in salmon pink to alert you to the situation.

When the price is missing, adding a price to the part in the Price text box on the record line will do two things.

  • First, it will make it possible actually to do the consignment processing for the item. (Without a price it cannot be done, either by itself or in conjunction with other records.)
  • Second, it will also automatically update the customer's price list for the part.

Adding the price and specifying the PO Number, Used Quantity, and selecting the OK check box must all be done together. It is not possible to add the price only.

When all requirements are met, click the Process link. tcmIS will take several actions.

  • The Quantity On Hand will be decremented by the amount that had been entered into the Used Quantity text box.
  • A billable usage transaction for the selected item(s), in the appropriate billing period, is created.
  • If the Inventory Type is MM and the remaining Quantity On Hand falls below the minimum, automatic replenishment will be triggered.
  • A Distribution MR is created and the material is 'delivered' (pick, pick QC, and ship confirms) from the consigned inventory group to the customer.
    • These MRs can be viewed on the MR History page after processing on the Consignment Processing page.

Click the Create Excel button to export the report in Excel format.
