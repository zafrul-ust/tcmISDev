+ Larger Font | + Smaller Font

The EDI Order Status page is used to view orders sent to tcmIS electronically, which are not able to be processed and are therefore stuck in error status. Electronic Data Interchange, or EDI, refers to the structured transmission of orders from our customers to Haas TCM via electronic means. It is used to transfer electronic documents from one computer system to another. The EDI Order Status page can be accessed from the Supply Chain Menu. Under Purchasing, select EDI Order Status. The page is launched in a new tab.

Figure 1

Select a customer from the Company drop-down box and click Find to display all EDI orders currently stuck in error status. You may sort by any of the columns displayed by clicking on the column name.

Figure 2

If you have the correct permissions, you may resolve the error by changing the quantity, the unit of measure, or the ship to location on the order. Figure 2 shows the EDI Order Status page where the user has view only permission. Figure 3 shows the EDI Order Status page where the user has both permission to change the ship to location and permission to change the order quantity and unit of measure. Contact the Technical Center to request permissions.

Check to box in the Reset column for all lines where the error has been corrected and click the Reset Status link above the data grid. TcmIS will try to process the order again. Check the EDI Order Status page later to see if the error was resolved.

Verify with the customer that this change is acceptable prior to resetting the order. Contact the Catalog Team or the Technical Center if unsure how to resolve the error.

There are many different error categories. A short explanation of the most common and possible solutions is below

ITEM QTY NOT INTEGER- The customer has orders in a unit of measure and quantity that does not equal a full container. Adjust the quantity on the order or enter a new chemical add request for a new packing size.

UNK PART NUMBER - The customer part number on the order does not exist in tcmIS. This could be because Haas has multiple items defined for the same customer part number. In this case, click the Multiple link in the Part Num column and select the correct item. Or submit a catalog add request to add the new material to the catalog.

UOS MISMATCH - The customer's unit of measure on the order does not exist in tcmIS as a valid UOM for the customer part number being ordered. Select a valid UOM (it may be necessary to change the quantity on the order). or contact the Catalog Team to add a new UOM and conversion factor.

BAD SHIPTO - The ship to location sent on the order does not exist in tcmIS. Either select a valid ship to location and reset the order or contact the Technical Center to add a new ship to location.
