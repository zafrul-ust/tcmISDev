Scanning Overview

Scanning process overview

See, also, Scanning Tips.

  • Barcode scanners are used to inventory (count) items/parts.
  • Each Work Area and Bin is identified with a unique barcode label as is each item.
  • A list of work area and part IDs are loaded into barcode scanners from tcmIS.
  • After counting, the barcode scanner is used to upload the inventory data to tcmIS.
  • tcmIS calculates replenishment requirements.
  • tcmIS creates material requests as required.

Logging into the Scanner

Note: The scanner must be in the cradle to log in.

  • Enter your tcmIS login ID in the User ID box.
  • Enter your tcmIS password in the Password box.
  • Select your company from the Company list. (Most users have only one choice here.)
  • Press the Login button on the screen to complete your log in.

When you are successful you will see the Main Page as illustrated below on the right.

  • If you unintentionally log out you can login again with the identical User ID and password without returning the scanner to the cradle.

On the Main Page you will see:

  • Three buttons, for the three major scanning activities:
    • Click the Load button to display the Work Area loading screen.
    • Click the Scan button to display the Work Area scanning screen.
    • Click the Submit button to display the Send Scans screen.
  • A Menu link to display the main menu.
  • Information
    • The currently logged in user name.
    • The date and time the work area data was last loaded.

Clicking the Exit button on the Log in page cancels your log in and exits the TCMScan application.

Whether you Exit or not, the login ‘session’ expires after 12 hours. It is a ‘silent’ exit: there is no notification and the scanner will continue to read bar code labels. But, it will not upload or download data to or from tcmIS until you log in again.

Loading Work Areas

Note: The scanner must be in the cradle to load work area information.

On the Main page (Figure 8A, above), click the Load button.

The facilities list will display all of the facilities that the user has permission to scan. Most users will see only one facility.

Select your facility.

  • Select the check box, or
  • Use the up and down arrow keys. Press the space key when your facility is highlighted.

Click the Load button and the work area information will be updated from tcmIS.

When the data load is finished a confirmation message is displayed; select ‘OK’ to return to the main screen.

It is not necessary to Load the scanner before every scan activity but it should be done at least once a day to keep the data current.

Selecting the Cancel button will return to the main screen without loading work areas.

Replenishment Scanning

Taking Inventory is organized in a hierarchy. At the highest level is the work area, often a physical cabinet. Every work area includes one or more sections, often a shelf divider or bin. Sections contain material (parts, items) to be counted.

In scanning, when an item is scanned, TCMScan will select the needed counting method:

  • Kanban style;
  • counted by item; or,
  • counted by receipt.

The scanner will display the appropriate count type prompts for each item when it is scanned.

Part ID Error Messages

If you receive an error, please read through the following troubleshooting suggestions.

Invalid Label.

  • Caused when the label scanned is not a part ID label.
  • Rescan the part label to continue.

This part has been made inactivate.

  • Caused when the part has been made inactivate in tcmIS.
  • Do not count this part, it may have material that is being used up but it cannot be replenished because of its inactive status.

This part has not been configured for this work area.

  • Caused when a part has been moved to a different work area without being relabeled.
  • The first 8 characters of the part ID represent the work area ID.
  • Return the part to its proper work area, or
  • Verify the tcmIS record is updated with the proper work area ID and replace the part ID label.

This part has not been configured.

  • Caused when the work area ID scanned does not match any of the data loaded on the scanner.
  • If the part was configured after the scanner was loaded last, reload the scanner and try again.

Counting methods

Kanban counting method

KanBan is a counting method that is a variant on MinMax where the reorder point is 1 “bin” and the maximum is 2 bins. Each “bin” contains a part-specific quantity which is specified as part of the catalog process.

The counting responsibility is to identify how many bins are needed.

  • If the bin quantity on hand is 0, 2 bins are needed: select 2.
  • If the bin quantity on hand is 1, 1 bin is needed: select 1.
  • If the bin quantity on hand is 2, no replenishment is required: select 0.

TCMScan will advance to the next prompt as soon as a quantity is selected.

Figure 14

Item counting method

Item Counting is exactly what it says. The counting responsibility is to identify how many items are on hand.

Count the number of containers in the part location. Enter the count in the quantity box, then select next.

Occasionally a part location will be configured for more than one item. An example would be a part location that is configured for an item that is active as well as an obsolete item that has not yet been depleted. In these cases it is important to count each item separately.

  • Every container is labeled with a tcmIS receipt ID label. Use the Item ID on the receipt label to group the containers by item for counting.
  • Select the Item ID from the dropdown list, enter the quantity count for that item, select the Apply Qty button to update the Item Quantity grid.
  • Do this for each different Item ID in this part location.
  • After all items have been entered select Next to advance.

Focus

The setup screen is opened by selecting Setup from the main Menu. The Focus Setting defaults to Auto. Auto focus is actually the rapid switching between near and far focus. When the focus is set to Auto the switching can be heard as a fluttering sound whenever the decode trigger is pulled. If you have trouble scanning barcodes try the other focus modes to see if they work better.

Figure 21

Beep

The default beep setting is Soft. When set to Soft, the beep volume can be adjusted with the volume keys. If the beep is difficult to hear try the Loud setting. When set to Loud, the beep is significantly louder and is not adjustable.

Figure 22

Timeout

The default timeout setting is 4000mS (4 seconds). If you have timeout errors while receiving or sending data to tcmIS increase this setting. The Set button tests the time it takes to receive a reply from tcmIS and sets the timeout value for 1.5 times that amount.

Figure 23

Expiration Threshold

Note: This feature is only available when the receipt label uses a 2D barcode which includes the expiration date.

The expiration threshold sets the number of days prior to expiration that the scanner will notify the user of the pending expiration. The notification occurs when today’s date is greater than the expiration date minus the Expiration Threshold.

Example: Today = April 10, 2012
  Expiration date = April 20, 2012
  Expiration Threshold = 14 days
  April 20 – 14 days = April 6

If the item is scanned after April 6 the scanner will notify the operator that the material will expire soon. If the Warning option is chosen the operator can choose to count the item. If the Error option is chosen the scanner will not count the item.

Figure 24

The Expiration Threshold setting works together with the ‘Pulled Within Days’ Work Area setting in tcmIS as set on the Work Area Definition page. Pulled Within Days is the number of days before an item expires within which it is to be removed from the work area; the scanner will not count the item as being still in inventory during this period. If this setting is greater than or equal to the Expiration Threshold, the Expiration Threshold has no effect.

Figure 25

Export the data count

The Main Menu includes an Export item to create a file of count data that can be used outside of tcmIS. The files are plain text in CSV (comma separated values) format. There are two options:

  • Current: The current data is saved as scandata.csv. It contains the Part ID, Cabinet ID, Company, Count Type, Quantity, Receipt ID or Item ID, Scan DateTime, User ID, and Logon ID fields.
  • History: The history is saved as scanhistory.csv. It contains all the fields in scandata.csv as well as Submitted DateTime for all scans submitted in the previous 14 days.
